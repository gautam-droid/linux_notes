Day 1 - File and Directory Operations

  1. cd ~ or cd : to go to home
  2. cd / : to go to root
  3. tree / : to display tree-like structure from root
  4. tree folderName : to display tree-like structure from folderName
  5. pwd : to print current working directory

  6. ls -l : list directory in detailed manner; similar commands related to ls are:

    • ls /home
    • ls /etc
    • ls /bin

    Note:

    • /home: user dirs are stored
    • /bin: binary files are stored that need to be available for all users
    • /etc: system config files are stored

  7. mkdir : to make a dir

  8. mkdir -p ~/project/practice/subdirectory : The -p option allows us to create parent directories if they don't exist. This command creates a practice directory inside your project directory, and a subdirectory inside practice.
  9. cd .. or cd - : to go back one level or move up one level in dir tree or previous directory
  10. mkdir dir1 dir2 dir3 : to create multiple directories

Creating a File

  1. touch file1.txt : to create an empty file or update the timestamp of existing files
  2. echo "Hello" > file.txt : echo prints text (if used alone), and > redirects the text into a file, creating the file if it doesn't exist.
  3. cat file.txt : to view the contents of a file
  4. echo "This is a new line" >> file.txt : >> appends to the file instead of overwriting it.

Copying, Moving, and Renaming Files

  1. cp file.txt dir1/ : copy file1.txt to dir1
  2. cp file.txt dir1/file_copy.txt : copy and rename the file in dir1
  3. mv file.txt dir2/ : moves the file into dir2
  4. mv file.txt newFileName.txt : renames the filename
  5. cp -r dir3 dir2/ : -r option tells cp to copy dirs recursively (including all subdirectories and files)

Note:
While moving/copying, note that the directory is present inside a particular working dir. If not present, then use the relative path.
For example:
If I'm inside dir1 and want to move a file to dir3, and if dir3 is not present inside dir1, then I have to use the command:
bash mv fileName.txt ../dir3
Here, I have used the relative path.

Viewing and Editing Files

  1. cat <<anyNameofDelimeter fileName.txt
    The content goes here
    and here too some content
    anyNameofDelimeter
    A here-document (<<) allows creating files with multiple lines of predefined content, using a custom delimiter, which is then redirected into a file.

  2. cat fileName.txt
    To view the contents of a file.

  3. nl fileName.txt
    To view the contents of a file with the line numbers.

  4. nano multiline.txt
    To edit the file using nano, a simple text editor.

Finding Files in Linux

  1. find . -name "*.txt"
    To find all text files in the current directory.

  2. sudo find / -name "passwd"
    To find the file named "passwd" starting from the root directory (/). sudo is used to elevate permissions since many system directories are not readable by regular users. So, sudo allows us to search these protected areas.

  3. find ~ -size +1M
    To find any file larger than 1MB.
    Note: We can use the - sign for negative size, meaning files smaller than the respective size.

  4. find ~ -mtime -1
    To search for files modified within a certain time frame. Here, -1 represents files modified within the last day.

  5. which python To find the location of exe files
